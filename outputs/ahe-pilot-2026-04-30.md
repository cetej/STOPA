## 2. Eight STOPA-extractable patterns

### Pattern 1 ⭐⭐⭐ — `change_manifest.json` schema (Decision Observability)

Z `evolve_prompt.md` lines 200-218. **Přímý copy-paste do STOPA `/self-evolve` outputs**:

```json
{
  "iteration": N,
  "changes": [
    {
      "id": "chg-1",
      "type": "new|improvement|rollback",
      "description": "What was changed and why",
      "files": ["relative/to/workspace/file.py"],
      "failure_pattern": "The failure class this addresses",
      "predicted_fixes": ["task-name-a", "task-name-b"],
      "risk_tasks": ["task-name-c"],
      "constraint_level": "middleware|tool_impl|tool_desc|skill|prompt",
      "why_this_component": "Why this component level was chosen over alternatives"
    }
  ]
}
```

**STOPA gap**: `optstate/<skill>.json` má `change_ledger[]` (max 20 FIFO) ale **nemá** `predicted_fixes` + `risk_tasks` + `why_this_component`. Predikce není zaznamenána ⇒ nelze měřit calibration drift.

**Action**: rozšířit `change_ledger[]` schema v `.claude/rules/memory-files.md` § Optimization State o tři nová pole. Backward compatible (default empty arrays).

### Pattern 2 ⭐⭐⭐ — Component-level taxonomy (Component Observability)

8 component types z `evolve_prompt.md` lines 86-95:

| Component | STOPA má? | Gap |
|-----------|-----------|-----|
| System Prompt | ✅ | — |
| Tool Description | ⚠️ částečně | description in skill body, ne separate file |
| Tool Implementation | ✅ | scripts/, hooks/ |
| Middleware | ❌ **GAP** | STOPA nemá middleware vrstvu (jen hooks které jsou různé) |
| Skill | ✅ | — |
| Sub-Agent | ✅ | agents/ adresář |
| Long-Term Memory | ✅ | learnings/, key-facts.md |
| Short-Term Memory | ⚠️ | intermediate/ má 24h TTL — ne přesně match |

**STOPA gap**: middleware vrstva (intercepts/transforms execution layer). STOPA hooks fire na PreToolUse/PostToolUse, ale **wrap_model_call** (intercept LLM calls) chybí.

**Action**: research, jestli STOPA potřebuje middleware vs. hooks postačí. Pravděpodobně postačí — STOPA nemá frequent need pro context compaction at runtime. Note jako future capability.

### Pattern 3 ⭐⭐⭐ — KEEP/IMPROVE/ROLLBACK+PIVOT attribution

Z `evolve_prompt.md` lines 172-176. Decision rule pro iteration 2+:

- **KEEP**: working, leave as-is
- **IMPROVE**: directionally correct, refine
- **ROLLBACK + PIVOT**: not working at this component level → rollback + re-approach **at different component level**

**Anti-pattern flagged explicitly**: "If the same failure class persists across 2+ iterations despite fixes at one component level, that level may be the wrong choice."

**STOPA gap**: `/self-evolve` má rollback při score regression, ale **nemá explicit PIVOT mechanism** (try different component type). Když se např. prompt edit nepovede 2× po sobě, STOPA neeskaluje na "zkus middleware/tool".

**Action**: extend `/self-evolve` Phase 6 (κ Commit) — když failure persists 2+ iterací na stejné failure pattern, force component-level switch v Phase 3 (σ Select).

### Pattern 4 ⭐⭐ — Best-of-N s explicit strategy constraints

Z `base.yaml` lines 39-44:

```yaml
best_of_n:
  enabled: false  # default off, but architecture supports it
  n: 2
  strategy_hints:
    - "You MUST focus on STRUCTURAL changes: middleware, tool implementations, or sub-agents.
       Do NOT modify system prompt or skill content this iteration."
    - "You MUST focus on GUIDANCE changes: system prompt rules, skill packages, tool descriptions,
       or LongTermMEMORY. Do NOT create or modify middleware Python files this iteration."
```

**Klíčový insight**: 2 variants běží paralelně s **mandatory orthogonal constraints**. Cross-variant comparison = signal pro next iteration.

**STOPA má**: orchestrate farm tier (5-8 agents parallel) + Cursor 3.0 `/best-of-n` reference (radar.md line 304).

**Gap**: STOPA farm tier nemá orthogonal strategy constraints — wraps často konvergují na stejné approach.

**Action**: do `/orchestrate` farm tier přidat `strategy_constraints[]` parameter. Ale optional — research-only insight, žádný immediate hook.

### Pattern 5 ⭐⭐⭐ — Pre-built `analysis/` tier (Experience Observability)

Z `evolve_prompt.md` lines 162-167. **Mandatory order of consumption**:

1. `evolution_history.md` — cumulative iteration history
2. `runs/iteration_NNN/input/analysis/overview.md` — pre-analyzed root causes  ⬅ **PRIMARY**
3. `runs/iteration_NNN/input/analysis/detail/{task_name}.md` — per-task deep
4. **Fallback only**: raw `nexau_in_memory_tracer.cleaned.json`

**Quote**: "do NOT skip them to read raw traces directly"

**STOPA má**:
- Raw layer: `outcomes/*.md` per-run RCL records, `failures/*.md` HERA trajectories, `farm-ledger.md`
- Distilled layer: ❌ **CHYBÍ**

**STOPA gap**: žádná pre-built `analysis/overview.md` která agreguje recent outcomes/failures do queryable summary. `/self-evolve` Phase 2 (ρ Reflect) musí číst raw outcomes.

**Action #1 v doporučeném rolloutu** (původní plan). Build `scripts/evidence-corpus-build.py`:
- Input: scan `outcomes/`, `failures/`, recent `farm-ledger.md`
- Output: `.claude/memory/intermediate/analysis-overview.md` (Layer 1) + `.claude/memory/intermediate/analysis/detail/*.md` (Layer 2)
- 24h TTL (already STOPA convention pro intermediate/)
- `/self-evolve` Phase 2 reads Layer 1 first, drills only when needed

### Pattern 6 ⭐⭐ — Partial-pass diagnostic

Z `evolve_prompt.md` lines 179-181:

> "Partial-pass tasks (some rollouts pass, some fail) are the most valuable. Compare the passing and failing rollouts of the same task, find the divergence point, and make the successful strategy the reliable default."

**STOPA fit**: `/eval` skill replays harness traces. Mohlo by to detekovat partial-pass patterns automaticky.

**Action**: extend `/eval` Phase 3 (Replay analysis) — když multiple runs same task mají různé outcomes, generate "divergence point" report. Optional, low priority.

### Pattern 7 ⭐ — pass@1 jako single optimization target

Z `evolve_prompt.md` line 177:

> "**The sole optimization target is pass@1**. Every change you make should raise pass@1. Timed-out tasks count as failures... Only pure infrastructure exceptions can be ignored."

**STOPA fit**: STOPA nemá explicit pass@1 metric ale `successful_uses / uses` v learnings/ je analog (per-pattern utility ratio).

**Already covered**. No action needed.

### Pattern 8 ⭐ — Safety constraints (LLM Config Hands-Off Rule)

Z `evolve_prompt.md` lines 229-241. Klíčová: **"Do NOT modify LLM configuration"** (model, temperature, max_tokens, reasoning_effort).

> "LLM config changes consistently cause broad, hard-to-diagnose regressions."

**STOPA má**: `commit-invariants.md` invariants I1-I9. Žádný explicit "don't change LLM config" pravidlo.

**Action**: add I10 do `commit-invariants.md` — "Do not modify model/temperature/reasoning_effort settings v skill frontmatter during iterative skill evolution. These are operator decisions, ne improvement targets."

## 6. Risks identified

| Risk | Mitigation |
|------|-----------|
| AHE Decision Observability "cargo cult" — agent píše predictions just to satisfy gate | Calibration tracking (post-hoc actual vs predicted match rate) sám detekuje pattern matching |
| Evidence corpus tier 1+2 dupluje information v outcomes/ | Single source of truth: corpus je generated VIEW, ne authoritative — original outcomes/ zůstávají |
| Component-level PIVOT vede k thrashing (try every level) | Hard limit: max 1 PIVOT per failure pattern; po 2 PIVOTs eskaluj user |
| AHE optimizuje na specific evals (Terminal-Bench, SWE-bench) — STOPA má vlastní | Patterns platí, čísla NE. Žádný transfer claim, jen methodology adopcí |
