# Upgrade panic-detector.py with doom-loop signal (ml-intern port)

## Context

`/radar` evaluated huggingface/ml-intern (2026-04-21, 🟡 7/10) and identified `agent/core/doom_loop.py` as directly adoptable into STOPA's `.claude/hooks/panic-detector.py`.

Current panic-detector.py detects 5 signals (edit velocity, bash fails, edit-fail cycles, scope creep, confused re-reads). Blind spot: "same tool called 3× with identical args" and "[A,B,A,B] tool-call sequences". Example gap: Claude repeatedly calls Grep with the same pattern expecting different results — no existing signal catches this.

Reference implementation: https://github.com/huggingface/ml-intern/blob/main/agent/core/doom_loop.py
- `extract_recent_tool_signatures()` — lookback 30, returns [(tool_name, args_hash)]
- `detect_identical_consecutive(sigs, threshold=3)` — 3+ consecutive identical → tool_name
- `detect_repeating_sequence(sigs)` — [A,B,A,B] length 2-5, 2+ reps → pattern
- Args hash: `hashlib.md5(args_str.encode()).hexdigest()[:12]`

## Scope (STRICTLY these 3 changes — no other refactoring)

### 1. Add `args_hash` to event dict in `extract_event()`
- Hash full tool_input JSON via `json.dumps(tool_input, sort_keys=True)` → MD5 → `[:12]`
- Store as `event['args_hash']`
- Keep existing `file` / `cmd_prefix` fields — they're used by signals 4 & 5

### 2. Add Signal 6: doom_loop to `compute_signals()`
- Build signatures from `scored_events` (reuses existing infra-error exclusion): `[(e['tool'], e['args_hash']) for e in scored_events]`
- If `detect_identical_consecutive(sigs, threshold=3)` → `+3 pts`, signal `f'doom_identical:{tool_name}'`
- Elif `detect_repeating_sequence(sigs)` → `+2 pts`, signal `f'doom_sequence:{A->B}'`
- Port two helpers as module-level functions operating on `list[tuple[str, str]]` (skip the ToolCallSignature dataclass wrapper — unnecessary)

### 3. Tailored intervention message at yellow/red threshold
- If `doom_identical:<tool>` in signals → append: "Pattern: 3× stejný toolcall na '{tool}' se stejnými argumenty. Změň přístup nebo se zeptej uživatele."
- If `doom_sequence:<A->B>` in signals → append: "Pattern: opakující se sekvence {A→B}. Zacyklil ses — zkus jiný přístup."
- Keep existing yellow/red message skeleton — just append the pattern-specific line

## Hard rules (constraints)

- **NO refactor** of signals 1-5, thresholds (4/6/7), cooldowns (180s/300s), or structured/exploratory gating
- **Update yellow failure-signal gate** (line 484): add `'doom_'` to allowed prefixes → `any(s.startswith(('bash_fails', 'edit_fail_cycle', 'doom_')) for s in signals)`. Rationale: doom-loop is valid panic even without bash failures (repeated Grep/Read).
- **Size budget**: current 526 lines → target ≤ 610 lines (+15% per invariant I2). If over, move helpers to `.claude/hooks/lib/doom_detector.py` (per hooks/CLAUDE.md lib/ convention).
- **Windows-compatible**: pathlib, UTF-8, no subprocess
- **Performance**: <50ms hook budget; MD5 on short JSON is microseconds — fine
