# AI Assistance and Coding Skill Atrophy

**Source**: Anthropic Research  
**Study**: AI-assisted vs hand-coded groups, comprehension quiz + debugging measurements

## Core Finding

AI accelerates task completion but undermines skill acquisition. Users who delegated wholesale to AI scored ~17% lower on comprehension quizzes (50% vs 67%) than hand-coders. **Debugging questions showed the largest performance gap.**

## Two Modes of AI Use

| Mode | Outcome | Behavior |
|------|---------|----------|
| **Strategic** | Strong mastery | Ask explanations, follow-up questions, then code independently |
| **Delegative** | Skill atrophy | Wholesale code delegation without problem-solving engagement |

## Oversight Capability Paradox

As coding gets automated, humans still need debugging ability and conceptual understanding to validate AI output — but these are precisely the capabilities that atrophy through over-reliance. The more you delegate, the less capable you become at overseeing what you delegated.

## Speed Results

Speed improvements from AI assistance were **minimal and statistically insignificant** in the study. The perceived productivity gain may not reflect actual throughput for learning contexts.

## Optimal Pattern

AI for conceptual clarification + coding independently = strongest mastery. Using AI to understand *why* (not just *what*) preserves skill formation.

## STOPA Relevance

This finding applies to how STOPA orchestrates work: skills that do too much for the user (complete implementations without explanation) may erode the user's system understanding over time. Skills should prefer exposing reasoning and offering checkpoints over silent autonomous execution. Connects to the autonomy-friction tension: high autonomy optimizes short-term throughput, strategic-mode interaction optimizes long-term capability.
