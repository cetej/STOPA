# SWE-Adept

## Authors
Kang He, Kaushik Roy

## Key Concepts
- Dual-agent architecture: localization agent + resolution agent
- Agent-directed depth-first search (ADFS) for code dependency traversal
- Checkpoint-indexed working memory (Git-based)
- Branching/reverting for alternative solutions
- Specialized progress tracking tools

## Main Claims
Existing repo-level SWE agents struggle with deep codebase navigation and effective context management. SWE-Adept separates localization (find relevant code) from resolution (apply fixes) so each agent has tailored context discipline.

## Core Findings
- +4.7% improvement over prior methods on SWE-Bench Lite + SWE-Bench Pro
- Localization agent uses ADFS, not BFS — minimizes context window clutter
- Shared working memory indexed by execution steps allows precise revert/branch
- Adaptive planning in resolution agent reduces redundant edits
