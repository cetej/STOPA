# How AI Assistance Impacts the Formation of Coding Skills

## Key Concepts
- Cognitive offloading — reduced engagement when delegating to AI
- Skill atrophy — competency decay through reduced hands-on practice
- Strategic vs delegative AI use — follow-up questions vs wholesale delegation
- Oversight capability paradox: need understanding to validate AI output
- Learning mode determines retention, not just task completion

## Main Claims
While AI accelerates task completion, this efficiency may undermine skill acquisition. The way you interact with AI while trying to be efficient affects how much you learn. As coding gets automated, humans still need debugging abilities and conceptual understanding to validate AI output — capabilities that atrophy through over-reliance.

## Core Findings
- AI users scored ~17% lower on comprehension quizzes than hand-coders (50% vs 67%)
- Debugging questions showed the largest performance gap between groups
- Speed improvements were minimal and statistically insignificant
- High-learning outcome: strategic use (ask explanations, follow-up questions)
- Low-learning outcome: heavy reliance (wholesale code delegation without problem-solving)
- AI for conceptual clarification + coding independently = stronger mastery
