# CoCR-RAG: Concept-Oriented Context Reconstruction

**Paper**: arXiv:2603.23989

## Problem

Retrieved web documents from multiple sources have inconsistent formats and styles. "Fusing such multi-source documents into a coherent and knowledge-intensive context remains a significant challenge."

## Solution: CoCR-RAG Framework

Three-stage pipeline:

### 1. Concept Distillation
Extracts essential concepts from Abstract Meaning Representation (AMR) — AMR represents text semantics as logical graphs for structured knowledge extraction.

### 2. Unified Context Reconstruction
LLMs synthesize distilled concepts into cohesive contexts by supplementing only necessary sentence elements. Avoids direct document concatenation.

### 3. Plug-and-Play Integration
Works across different LLM backbones — adaptable component, not model-specific.

## Results

- Significant performance improvements on PopQA and EntityQuestions datasets
- Outperforms existing multi-source RAG methods
- AMR-based extraction is linguistically grounded (not surface-level heuristics)

## Key Insight

Reconstruct context from concepts, not from raw document text. AMR as intermediate semantic layer removes style/format noise while preserving knowledge structure.
