# Headless Everything for Personal AI

**Source**: simonwillison.net, April 19, 2026 — Simon Willison

## Main Thesis

"Headless" services—accessed via APIs rather than GUIs—are becoming standard because AI agents interact more efficiently with APIs than navigating UIs. Matt Webb proposes that "personal AIs is a better experience for users than using services directly" and that "headless services are quicker and more dependable for the personal AIs."

## Key Entities

- **Matt Webb**: Coined the headless services trend
- **Marc Benioff / Salesforce**: Announced "Salesforce Headless 360" — exposing platform via APIs, MCP, and CLI
- **Brandur Leach**: Predicted APIs as decisive competitive differentiators in enterprise software

## Technical Details

- Salesforce Headless exposes data, workflows, and tasks through APIs, MCP, and CLI
- Integration channels: Slack, voice interfaces, API endpoints
- Direct agent access replaces browser automation

## Business Implications

- Threatens traditional SaaS per-seat pricing models
- API availability → decisive competitive factor
- Shifts value from "good UI" to "good API surface"
