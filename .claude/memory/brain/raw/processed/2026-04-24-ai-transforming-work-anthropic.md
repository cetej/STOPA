# How AI Is Transforming Work at Anthropic

## Key Concepts
- Paradox of supervision: Claude effectiveness requires human oversight, but oversight skills may atrophy
- Selective delegation: easily verifiable, outside expertise, low complexity, well-defined, boring tasks
- Full-stack capability expansion — engineers tackle broader scope with Claude
- "Manager of AI agents" identity shift
- 27% new work that wouldn't otherwise occur

## Main Claims
Claude usage intensified dramatically (28% → 59% of daily work in one year). Self-reported productivity gains doubled (20% → 50%). Engineers are developing intuitive delegation strategies — keeping design/strategic thinking in-house, delegating verifiable/defined tasks. Career uncertainty is widespread despite short-term optimism.

## Core Findings (Survey: 132 respondents, Interviews: 53, Claude Code: 200K transcripts)
- Daily debugging usage: 55%
- Task complexity on CC: 3.2 → 3.8 (five-point scale)
- Consecutive tool calls without intervention: 9.8 → 21.2 (doubled)
- Human interventions per transcript: 6.2 → 4.1 (−33%)
- Feature implementation tasks: 14% → 37% of usage
- Design/planning tasks: 1% → 10% of usage
- ~27% of Claude-assisted work = entirely new work
- Most engineers fully delegate only 0-20% of tasks

## Task Delegation Heuristics
Engineers delegate when task is: easily verifiable, outside personal context, low complexity, well-defined scope, "boring" work. Never delegate: design decisions, strategic thinking.

## Tensions
- Productivity gains ↔ skill erosion concerns
- Full-stack expansion ↔ long-term role uncertainty
- Reduced collaboration friction ↔ lost mentorship/peer-learning
