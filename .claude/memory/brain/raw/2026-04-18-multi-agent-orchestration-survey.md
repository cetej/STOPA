## Summary

Survey and framework paper presenting unified architectural layer for orchestrating multi-agent systems, integrating planning, policy enforcement, state management, and quality operations. Describes two key protocols: Model Context Protocol (MCP, standardizing agent access to external tools/context) and Agent2Agent protocol (peer coordination, negotiation, delegation). Addresses governance, observability, and accountability for enterprise-scale AI deployment. Architectural and taxonomic contribution, no benchmarks.

## Key Concepts

- Unified orchestration layer: planning + policy + state + quality
- Model Context Protocol (MCP): standard for tool/context access
- Agent2Agent protocol: peer coordination, negotiation, delegation
- Governance and observability for enterprise AI
- Accountability mechanisms in multi-agent systems

## Claims

- MCP standardizes agent-tool interface (adopted by Anthropic, others)
- Agent2Agent protocol enables structured peer-to-peer coordination
- Enterprise adoption requires governance + observability stack beyond research architectures
