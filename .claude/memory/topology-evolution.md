# Topology Evolution (HERA-inspired)

Tracking orchestration topology per run to detect emergent patterns.
HERA source: arXiv:2604.00901 — agent networks spontaneously reorganize as experience accumulates.

## How to use

- Orchestrator writes a snapshot after each completed orchestration (Phase 6)
- `/eval --topology` analyzes trends after 20+ entries
- Data informs default tier/agent selection per task_class

### Column Definitions

- **Agents**: Number of sub-agents used
- **Node Eff**: Result quality / agent count (PASS=1.0, WARN=0.5, FAIL=0.0 / agents)
- **Retries**: Total retry count across all agents
- **Critic Loops**: Number of critic FAIL→fix→critic cycles
- **Result**: PASS / WARN / FAIL
