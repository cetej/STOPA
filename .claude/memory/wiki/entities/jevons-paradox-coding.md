# Jevons Paradox Applied to Coding

> When code generation becomes cheaper via AI, demand for software increases enough to offset or exceed the savings — resulting in more engineering work, not less.

## Key Facts

- Classic example: ATMs made bank branch operation cheaper → more branches opened → more tellers hired (not fewer) (ref: sources/karpathy-nopriors-autoagent-loopy-era.md)
- Applied to coding: software was scarce because expensive; AI reduces cost → unlocks latent demand for software that wasn't viable before (ref: sources/karpathy-nopriors-autoagent-loopy-era.md)
- Karpathy's prediction: "demand for software will be extremely large" because code is now ephemeral, modifiable, not bound to existing imperfect tools (ref: sources/karpathy-nopriors-autoagent-loopy-era.md)
- Counterpoint: long-term displacement still possible; Karpathy notes this as "cautiously optimistic view, very hard to forecast" (ref: sources/karpathy-nopriors-autoagent-loopy-era.md)
- Implication: large-scale agent systems (like STOPA) create rather than destroy software demand (ref: sources/karpathy-nopriors-autoagent-loopy-era.md)

## Relevance to STOPA

Validates STOPA's bet on agent orchestration as a productivity multiplier: more capable agents → more projects attempted → more orchestration needed, not less. Guides ROI framing when presenting STOPA value to users.
