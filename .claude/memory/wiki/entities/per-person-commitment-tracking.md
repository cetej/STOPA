# Per-Person Commitment Tracking

> One markdown file per team member/contact tracking open commitments in both directions — what they owe and what is owed to them.

## Key Facts

- One file per person, updated after each meeting via automated note processing (ref: sources/chief-of-staff-openclaw.md)
- Tracks: commitments made by them, commitments made to them, last touchpoint, context from last meeting (ref: sources/chief-of-staff-openclaw.md)
- "If I asked someone to do something three weeks ago and there's no update, the system knows because it's in their file" (ref: sources/chief-of-staff-openclaw.md)
- Part of broader relationship CRM: also includes relationship history, what they care about, pipeline stage for LPs (ref: sources/chief-of-staff-openclaw.md)
- Compounds over time: pre-meeting brief pulls the file, post-meeting updates it (ref: sources/chief-of-staff-openclaw.md)

## Relevance to STOPA

STOPA has no relationship/CRM memory pattern. Per-person files in `memory/people/` would enable commitment tracking across NG-ROBOT contacts, board members, etc. Straightforward extension of existing memory file pattern.
