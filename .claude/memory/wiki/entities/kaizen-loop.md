# Kaizen Loop

> Weekly automated improvement cycle: cron research scan + community pattern mining + self-improvement suggestions from daily interaction friction.

## Key Facts

- Friday cron: AI scans community forums, checks new patterns, saves findings to `memory/kaizen-research-YYYY-MM-DD.md` (ref: sources/chief-of-staff-openclaw.md)
- Sunday review session: AI summarizes week's research, surfaces top ideas, human decides what to implement (ref: sources/chief-of-staff-openclaw.md)
- Also learns from daily interactions: repeated corrections, noisy filters, format friction → captured in memory → surfaces as change proposals (ref: sources/chief-of-staff-openclaw.md)
- Drives continuous system refactoring: build feature → run 2-3 weeks → assess actual fit → cut or simplify (ref: sources/chief-of-staff-openclaw.md)
- Key property: "smaller system you trust > bigger system you route around" — kaizen enforces this discipline (ref: sources/chief-of-staff-openclaw.md)
- Differs from human EA: can simultaneously scan hundreds of builders AND cross-reference against your specific system (ref: sources/chief-of-staff-openclaw.md)

## Relevance to STOPA

STOPA has `/watch` (ecosystem scan) and `/evolve` (pattern graduation) but no formal weekly cadence that COMBINES external scan + internal friction detection. Kaizen Loop = `/watch` + `/evolve` + cron schedule + human review session fused into one rhythm.
