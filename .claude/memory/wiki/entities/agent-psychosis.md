# Agent Psychosis

> The state of perpetual overwhelm when agent capabilities unlock faster than you can absorb them — feeling nervous when tokens go unused, racing to maximize throughput, sense that mastery is always just out of reach.

## Key Facts

- Described by Karpathy as a recurring state post-December 2025 capability jump (ref: sources/karpathy-nopriors-autoagent-loopy-era.md)
- Triggers: seeing others on Twitter doing impressive things, idle subscription tokens, any agent wait time when parallelization is possible (ref: sources/karpathy-nopriors-autoagent-loopy-era.md)
- The flip side: everything feels like "skill issue" — capability is there, you just haven't found how to string it together (ref: sources/karpathy-nopriors-autoagent-loopy-era.md)
- GPU analogy: PhD students felt nervous when GPUs sat idle — now it's tokens not flops (ref: sources/karpathy-nopriors-autoagent-loopy-era.md)
- Peter Steinberg antidote: 10 repos tiled on monitor, each agent taking 20-minute tasks — "macro actions over your repository" (ref: sources/karpathy-nopriors-autoagent-loopy-era.md)

## Relevance to STOPA

Agent psychosis is the user state that STOPA's orchestration skills are designed to serve — minimize human bottleneck, maximize parallel throughput. The `/status` and `/budget` skills should surface token utilization as a first-class signal.
