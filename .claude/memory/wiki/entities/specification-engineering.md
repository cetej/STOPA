# Specification Engineering

> Třetí vrstva hierarchie kontextového řízení (arXiv:2603.09619v2): Context Engineering → Intent Engineering → Specification Engineering. "Whoever controls specifications controls scale."

## Key Facts

- arXiv:2603.09619v2: CE → IE → SE hierarchie pro multi-agent systémy (ref: sources/idea-file-research.md)
- Context Engineering: ovlivňuje co model čte (behavior level) (ref: sources/idea-file-research.md)
- Intent Engineering: ovlivňuje strategii agenta (goal level) (ref: sources/idea-file-research.md)
- Specification Engineering: ovlivňuje škálování systému (architecture level) (ref: sources/idea-file-research.md)
- Red Hat "Four Pillars": vibes → specs → skills → agents jako progresivní abstrakce (ref: sources/idea-file-research.md)

## Relevance to STOPA

STOPA operuje na všech třech vrstvách: SKILL.md = context engineering, CLAUDE.md = intent engineering, orchestrate skill = specification engineering pro multi-agent decomposition. Tato taxonomie poskytuje rámec pro kategorizaci STOPA artefaktů a jejich evoluce.
