# AI Planning Framework for LLM-Based Web Agents

> Shahnovsky & Dror (2026) — formalizace web agentů jako POMDP, taxonomie Step-by-Step/Tree Search/Full-Plan → BFS/Best-First/DFS, 5 trajectory metrik, 794-trajektorií dataset z WebArena.

## Key Facts

- Taxonomie tří architektur mapovaná na klasické search: Step-by-Step→BFS, Tree Search→Best-First, Full-Plan-in-Advance→DFS (ref: sources/ai-planning-framework-web-agents.md)
- Full-Plan má 89% Element Accuracy ale jen 36.29% task success vs Step-by-Step 82% / 38.41% — "agent perfektně provádí špatný plán" (ref: sources/ai-planning-framework-web-agents.md)
- 5 metrik: Recovery Rate, Repetitiveness, Step Success Rate, Partial Success Rate, Element Accuracy (ref: sources/ai-planning-framework-web-agents.md)
- Dataset: 794/812 WebArena tasks, single-annotator (autoři sami), žádné inter-annotator agreement (ref: sources/ai-planning-framework-web-agents.md)
- BFS/DFS taxonomie derivativní od Yao 2023 ToT; Best-First od Koh 2024 (ref: sources/ai-planning-framework-web-agents.md)
- Tree Search agent definován v taxonomii ale kvantitativně neporovnán v experimentech (ref: sources/ai-planning-framework-web-agents.md)

## Relevance to STOPA

Recovery Rate a Element Accuracy jsou koncepty přenositelné na STOPA eval traces; finding "100% subtask completion + verify FAIL = špatný plán, ne špatná execution" je diagnostická heuristika pro `/critic` a `/orchestrate` review.
