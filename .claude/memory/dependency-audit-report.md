### MEDIUM

#### Anthropic SDK lower bound mismatch (ADOBE-AUTOMAT, ZACHVEV)
- **ADOBE-AUTOMAT:** `anthropic>=0.39.0` — written when SDK was at ~0.39.x
- **ZACHVEV:** `anthropic>=0.40` — written when SDK was at ~0.40.x
- **Current installed (global):** `anthropic 0.85.0`
- **Risk:** Code written against 0.39/0.40 API may break on 0.85 due to breaking changes (e.g., thinking mode API changes documented in `2026-04-01-sonnet46-thinking-effort-breaking-change.md`, new tool use format). The lower bound doesn't protect against breaking changes in newer versions — it only specifies minimum.

- **Files:** `ADOBE-AUTOMAT/backend/requirements.txt`, `ZACHVEV/pyproject.toml`

#### Missing lockfiles (6 projects)
- **Affected:** STOPA, ADOBE-AUTOMAT (backend + frontend), ZACHVEV, MONITOR, GRAFIK
- **Risk:** Non-reproducible builds. Transitive deps (torch, transformers, etc.) may drift between installs. Particularly risky for ZACHVEV which has heavy ML deps (`torch>=2.0`, `transformers>=4.36`, `hdbscan`, `umap-learn`).
- **Recommendation:**
  - Python projects using uv: `uv lock` (like POLYBOT)
  - Python projects using pip: `pip freeze > requirements.lock` (like NG-ROBOT)
  - Node projects: `npm install` generates `package-lock.json` automatically
